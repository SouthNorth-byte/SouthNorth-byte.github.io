    primary可以解决什么问题

    南北 10/11/2020

    本文主要看一下bean标签中的primary属性可以解决什么问题。

    # 存在的问题

    # 案例一

    定义一个NormalBean，如下：

    public class NormalBean {
    public interface IService{} //@1
    public static class ServiceA implements IService{} //@2
    public static class ServiceB implements IService{} //@3
}

    上面代码很简单，@1：定义了一个接口IService，@2和@3创建了两个类都实现了IService接口。

    下面我们通过spring来定义ServiceA和ServiceB两个bean，配置文件（normalBean.xml）如下：

    <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
    http://www.springframework.org/schema/beans/spring-beans.xsd">
    <bean id="serviceA" class="top.southnorth.NormalBean$ServiceA"/>
    <bean id="serviceB" class="top.southnorth.NormalBean$ServiceB"/>
</beans>

    注意@1的代码，从spring容器中在容器中查找NormalBean.IService.class类型的bean对象，我们来运行一下看看效果

    发生异常了，错误中有一段提示比较重要，如下：

    org.springframework.beans.factory.NoUniqueBeanDefinitionException: No qualifying bean of type 'top.southnorth.NormalBean$IService' available: expected single matching bean but found 2: serviceA,serviceB
    1

    这个详细说出了错误原因：spring容器中定义了2个bean，分别是serviceA和serviceB，这两个bean对象都实现了IService接口，而用例中我们想从容器中获取IService接口对应的bean，此时容器中有2个候选者（serviceA和serviceB）满足我们的需求，此时spring容器不知道如何选择，到底是返回serviceA呢还是返回serviceB呢？spring容器也懵逼了，所以报错了。

    再来看一个通过setter方法注入的案例：

    public class SetterBean {

    public static class ServiceB implements IService{} //@3
    private IService service;
    public void setService(IService service) {
        this.service = service;

    <bean id="serviceA" class="top.southnorth.SetterBean$ServiceA"/>
    <bean id="serviceB" class="top.southnorth.SetterBean$ServiceA"/>
    <bean id="setterBean" class="top.southnorth.SetterBean" autowire="byType" />

    运行出错，错误中重点信息：

    org.springframework.beans.factory.NoUniqueBeanDefinitionException: No qualifying bean of type 'top.southnorth.SetterBean$IService' available: expected single matching bean but found 2: serviceA,serviceB
    1

    容器中去找IService接口对应的bean，期望有一个匹配的，实际上却找到了2个匹配的，不知道如何选择，报错了。

    上面2个案例报的异常都是下面这个异常：

    org.springframework.beans.factory.NoUniqueBeanDefinitionException
    1

    **结论：**当希望从容器中获取到一个bean对象的时候，容器中却找到了多个匹配的bean，此时spring不知道如何选择了，处于懵逼状态，就会报这个异常。

    # 解决方案

    spring中可以通过bean元素的primary属性来解决这个问题，可以通过这个属性来指定当前bean为主要候选者，当容器查询一个bean的时候，如果容器中有多个候选者匹配的时候，此时spring会返回主要的候选者。

    下面我们使用primary来解决上面案例的问题：

    public class PrimaryBean {
    public interface IService{} //@1
    public static class ServiceA implements IService{} //@2
    public static class ServiceB implements IService{} //@3
    private IService service;
    public void setService(IService service) {
        this.service = service;
    }
    @Override
    public String toString() {
        return "PrimaryBean{" +
                "service=" + service +
                '}';
    }
}

    bean xml配置文件如下：

    http://www.springframework.org/schema/beans/spring-beans.xsd">
    <bean id="serviceA" class="top.southnorth.PrimaryBean$ServiceA" primary="true"/>
    <bean id="serviceB" class="top.southnorth.PrimaryBean$ServiceB"/>
    <bean id="setterBean" class="top.southnorth.PrimaryBean" autowire="byType" />

    上面配置中我们将serviceA的primary属性置为true了，将其置为主要候选者，容器中如果查找bean的时候，如果有多个匹配的，就以A为主。

    我们来个测试用例：

    @Test
public void primaryBean() {
    String beanXml = "classpath:primaryBean.xml";
    ClassPathXmlApplicationContext context = IocUtils.context(beanXml);
    PrimaryBean.IService service = context.getBean(PrimaryBean.IService.class); //@1
    System.out.println(service);
    PrimaryBean primaryBean = context.getBean(PrimaryBean.class); //@2
    System.out.println(primaryBean);
}

    @1：从容器中查找IService类型匹配的bean，这个接口有2个实现类（ServiceA和Service2），这两个类在容器中都定义了，但是serviceA为主要的bean，所以这行代码会返回serviceA

    @2：从容器中查找PrimaryBean类型的bean，容器中有一个，这个bean按照byType默认注入IService接口匹配的bean，注入的时候如果候选者有多个，以primary=”true”的bean为主来注入。

    我们来运行一下，看看和分析的是否一致，运行输出：

    top.southnorth.PrimaryBean$ServiceA@7b227d8d
PrimaryBean{service=top.southnorth.PrimaryBean$ServiceA@7b227d8d}
    1
    2

    上面输出确实都是ServiceA，和我们分析的一致。

    还有当候选者中如果有多个bean都将primary置为true，此时spring还是会懵逼的，也会报错，不知道如何选择了。

    # 总结

    当从容器中查找一个bean的时候，如果容器中出现多个Bean候选者时，可以通过primary=”true”将当前bean置为首选者，那么查找的时候就会返回主要的候选者，否则将抛出异常。
